# המשך עבודה — קרא את זה קודם אם נפתחה שיחה חדשה

מסמך זה נועד לצ'אט Claude חדש שנפתח בגלל התנתקות. הוא מסביר מה הפרויקט, איך עובד תהליך העבודה השוטף על תוכן, ומה המצב הנוכחי בדיוק.

## מה זה הפרויקט

"ערבית מצילה חיים" — PWA (אתר שמתנהג כאילו הוא אפליקציה) שמלמד ערבית מדוברת (לבנטינית/פלסטינית) מבצעית לכוחות ביטחון: מחסומים, תשאולים, חיפושים, אינטראקציה עם אוכלוסייה. הבעלים היא דוברת/מומחית ניב, לא מתכנתת — כל שיחה איתה מתחילה מתוכן גולמי (עברית + תעתיק עברי, לפעמים גם כתיב ערבי), והתפקיד שלי הוא לנקות, לסווג, להכניס לקוד, ולשלוח לה מסמך אימות לפני כל דבר.

**הקובץ היחיד שמריץ את כל האפליקציה: `index.html`** (~18,500+ שורות, JS וניל, בלי build system). יש גם `sw.js` (service worker לעבודה אופליין).

## עקרון הסיווג — חשוב מאוד, נקבע אחרי תיקון ישיר של המשתמשת

כשהיא שולחת תוכן גולמי, יש לסווג לפי הכלל הבא:
1. **מילה בודדת** → מילון (`BUILTIN_DICT`, נטמע ל־`DICT`)
2. **רצף נרטיבי רב-שלבי** (התחלה-אמצע-סוף, דמויות, "סיפור") → **תרחיש חדש** ב־`BUILTIN_SCENARIO_DATA`
3. **משפט שלם עצמאי** שאינו חלק מסיפור ואינו מילה בודדת → **קטגוריה קיימת** (לא קטגוריה חדשה, לא תרחיש!) — הכי קרוב לנושא שלו מבין הקטגוריות שכבר יש באפליקציה.

**טעות שנעשתה בעבר ותוקנה**: בעבר הכנסתי משפטים בודדים ישר למילון בקטגוריות חדשות שהמצאתי. המשתמשת תיקנה בחריפות: "אני לא הצליחה להבין למה אתה מכניס למילון דברים ישר ולא לקטגוריות שקיימות במבצעי". מאז — תמיד קודם `grep` לבדוק אילו קטגוריות כבר קיימות, ולשבץ לשם.

## תהליך עבודה סטנדרטי לכל אצווה (batch) של תוכן

1. **בדיקת כפילויות** — grep על `index.html` לוודא שהתוכן לא קיים כבר.
2. **בניית מסמך סקירה HTML** — סקריפט Python בתיקיית scratchpad (ראה תבנית למטה) שמנקה תעתיק, מרכיב JSON, ומייצר עמוד HTML כהה עם RTL, שדות `<input>` לעריכה, מקובץ לפי `<h2>` קטגוריות, ובר תחתון קבוע עם כפתורי "📋 העתק הכל" ו"⬇️ הורד קובץ מתוקן".
3. **שליחה** ל־SendUserFile (status: normal, display: render כדי שתראה בתוך הצ'אט).
4. **המתנה לתשובה מאומתת** — היא שולחת בחזרה טקסט בפורמט `עברית | תעתיק | ערבית` (עם `===שם קטגוריה===` כמפריד בין קטגוריות/תרחישים).
5. **סקריפט הכנסה** — פרסינג, `clean_tr()` שוב (ליתר ביטחון), יצירת שורות JS, הכנסה ל־`index.html` באמצעות Edit ישיר או סקריפט Python שמאתר עוגן (anchor) בקובץ.
6. **אם זה תרחיש**: להוסיף גם ל־`NEW_SCENARIOS` (לתג "🆕 חדש") וגם ל־`SCN_CATEGORIES` (אריח ברשת — **אחרת התוכן בלתי נראה!** ראה "מוקש קריטי" למטה).
7. **בדיקת syntax** (node -c או דומה) — ראה למטה.
8. **הרצת בדיקות עשן**: `node tests/smoke.js` — חייב "✅ כל 20 הבדיקות עברו" (או יותר, אם נוספו בדיקות) לפני commit.
9. **bump ל־CACHE ב־`sw.js`** — למשל `'asl-v92'` → `'asl-v93'`. **קריטי**: בלי זה, שינויים לא מופיעים אצל משתמשים כי ה-service worker מגיש גרסה ישנה מהמטמון. זו הסיבה הישירה לתסכול "השינויים לא מופיעים / זה נתקע".
10. **git commit** (אחרי הגדרת identity! ראה למטה) **ו-push לשני מקומות**: לענף הפיצ'ר וגם ל-main.

### תבנית לסקריפט ניקוי תעתיק (`clean_tr`) — להעתיק תמיד:
```python
import re, unicodedata
def clean_tr(s):
    out=[]
    for c in s:
        if c=='ّ' or c=='ﹼ' or c=='ﹽ':
            k=len(out)-1
            while k>=0 and (unicodedata.combining(out[k])!=0 or out[k] in '׳״'): k-=1
            if k>=0 and 'א'<=out[k]<='ת':
                if not (k+1<len(out) and out[k+1]=='ּ'): out.insert(k+1,'ּ')
            continue
        if '؀'<=c<='ۿ': continue
        if 'ﭐ'<=c<='﻿': continue
        if c=="'": out.append('׳'); continue
        out.append(c)
    return re.sub(r'\s{2,}',' ',''.join(out)).strip()
def jq(s): return "'"+s.replace("'","\\'")+"'"
```
זה ממזג שאדה ערבית (ّ) לתוך האות העברית הקודמת כדגש (ּ), מנקה תווים ערביים תועים, וממיר גרש ASCII לגרש עברי (׳) — קריטי כי גרש ASCII שובר מחרוזות JS.

### מוקש קריטי: תרחישים בלתי נראים
ה־UI של רשת התרחישים מרונדר **אך ורק** מהמערך הקשיח `SCN_CATEGORIES`. הוספת תוכן ל־`BUILTIN_SCENARIO_DATA` **בלבד** לא מספיקה — התוכן קיים בנתונים אבל בלתי נראה ב-UI. זו הייתה תקלה אמיתית שהמשתמשת דיווחה עליה ("אני לא רואה את התרחישים"). התיקון: (א) להוסיף כל קטגוריית תרחיש חדשה גם ל־`SCN_CATEGORIES` (עם אייקון), (ב) יש רשת ביטחון קבועה ב־`renderScnGrid()` שמזהה אוטומטית כל `cat` שקיים בנתונים אך לא ברשימה, ומוסיפה קבוצת "🆕 תרחישים נוספים" — כרשת ביטחון בלבד, לא תחליף להוספה נכונה. (ג) יש בדיקת עשן ייעודית שמוודאת שכל `SCENARIO.cat` מופיע ב-`#scn-grid` המרונדר — **תמיד להריץ את חבילת הבדיקות אחרי הוספת תרחיש**.

## מבנה קוד — נקודות מפתח

- `BUILTIN_TASHAUL` — משפטים בודדים למקטע military (מבצעי), משובצים ל-`cat` קיים.
- `BUILTIN_DAILY` — כמו למעלה אבל למקטע daily (חדש יחסית, נוצר השנה כדי לתמוך במשפטי יומיום עצמאיים).
- `BUILTIN_SCENARIO_DATA` — כל שלבי כל התרחישים, `id:'bi-<slug>-<n>'`, `tags:'step<n>'`.
- `SCN_CATEGORIES` — הרשת הקשיחה שמרנדרת את מסך התרחישים (ראה מוקש למעלה).
- `NEW_SCENARIOS` — מערך שמות קטגוריות תרחיש שמקבלות תג "🆕 חדש" עד שנפתחות בפעם הראשונה (localStorage `scn_seen`). **שים לב**: יש היסטוריה של כפילויות שנוצרו מריצות חוזרות של סקריפטים — לבדוק `grep -c` שאין כפילויות אחרי הוספה.
- `GREETINGS_DATA` (`cat:'ברכות ונימוסים'`) ו־`PROVERBS_DATA` (`cat:'ביטויים ופתגמים'`) — **תבנית שונה!** אלה מערכים למקטע `section:'heart'` ("פותחים לב") שמוזנים ל-Supabase ע"י כפתור אדמין חד-פעמי (`addGreetings()`/`addProverbs()`), לא דרך המיזוג האוטומטי ב-`loadData()`. אם מוסיפים שורות לשם — הן ייכנסו לאפליקציה רק אחרי שהאדמין ילחץ שוב על הכפתור (או שצריך לוודא שה-autoSync כבר מכסה זאת).
- `loadData()` — טוען מ-Supabase ואז ממזג את כל מערכי ה-`BUILTIN_*` לתוך המערכים בזיכרון, כדי שהאפליקציה תעבוד גם אופליין וגם אם ה-backend ריק/שבור.
- `autoSyncBuiltins()` — רץ בכניסת אדמין, דוחף כל תוכן BUILTIN_* שעדיין לא ב-Supabase (כדי שיהיה ניתן לעריכה/הקלטה).
- בדיקת syntax מהירה: `node -e "require('vm').Script(require('fs').readFileSync('index.html','utf8').match(/<script>([\s\S]*)<\/script>/)[1])"` — או פשוט להריץ את חבילת smoke, שתיכשל עם שגיאת syntax ברורה.
