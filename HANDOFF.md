# HANDOFF — מסמך מסירה למפתח/ת

מטרת המסמך: לאפשר למפתח חדש להיכנס לפרויקט **בלי שלב Discovery יקר** — מה קיים,
מה רדום ואיך מפעילים, מה נשאר לעשות ובאיזה סדר. משלים את:
`README.md` (מבט-על) · `ARCHITECTURE.md` (מפת קוד) · `commercial/README.md`
(שכבת המסחר) · `commercial/MEDIA.md` (הגנת מדיה) · `tests/README.md` (בדיקות).

---

## 1. מה המוצר
PWA ללימוד ערבית מדוברת מבצעית (עברית↔ערבית+תעתיק+הקלטות), offline-first,
ללא חשבונות לומדים. תוכן מאומת ע"י מומחית. היעד העסקי: מכירת מנויים (B2C)
ובהמשך גופים (B2B) — ראו סקירת יועץ חיצונית שהוגדרה כ"פרויקט מסחור והקשחה".

## 2. מה קיים ועובד היום (production)
- אפליקציה חיה (Cloudflare Workers, דיפלוי אוטומטי מ-main).
- תוכן: ~1,700 שורות ב-Supabase + תוכן מובנה בקוד (מילון מבצעי 427 מילים
  ב-19 נושאים, 193 שלבי תרחישים) שמסתנכרן ל-DB אוטומטית תחת אדמין.
- אדמין מלא: עריכה, הקלטה (Cloudinary), תור הקלטה רציף, רשימת מילים ללא
  הקלטה, כלים חד-פעמיים.
- גיבוי: כל התוכן (JSON) + כל ההקלטות (ZIP+index.csv) + תזכורות 30 יום.
- מצב מאובטח (SECURE_MODE): אפס קריאות צד-ג' — לטובת מכירה לגופים.
- יומן שגיאות client-side (ring buffer) עם צפייה/הורדה באדמין.
- בדיקות עשן: `node tests/smoke.js` (19 בדיקות, ~20 שניות).

## 3. מה קיים אבל רדום (כתוב, לא מחווט)
| רכיב | איפה | סטטוס |
|---|---|---|
| התחברות טלפון+OTP | `index.html`, דגל `COMMERCE_LOGIN` | בנוי ונבדק מול OTP מדומה; מוסתר כברירת מחדל (`?login=1` מפעיל) |
| זכאות Offline | `clHasAccess`/`clRefreshEntitlement`/`clClearPremiumCache` | בנוי ונבדק; חסד 14 יום, רענון כל 3 ימים |
| סכימת מנויים + `is_subscribed()` | `commercial/01_schema.sql` | מוכן להרצה (בטוח) |
| תיוג free/premium | `commercial/02_content_tier.sql` | מוכן; **דורש החלטה עסקית** מה חינם |
| חומת תשלום (RLS) | `commercial/03_rls_content.sql` | מוכן; **להריץ אחרון** (ראו סדר) |
| Stripe webhook | `commercial/functions/stripe-webhook/` | מוכן לפריסה |
| חתימת אודיו למנויים | `commercial/functions/media-sign/` + `MEDIA.md` | מוכן; דורש Cloudinary authenticated |

## 4. עבודה שנשארה — לפי סדר מומלץ
1. **הפעלת Supabase Auth טלפוני** (ספק SMS ב-Supabase) ובדיקת מסך ההתחברות
   מול השרת האמיתי (עד היום נבדק מול stub בלבד).
2. הרצת `01_schema.sql` + `02_content_tier.sql` (בטוחות; לערוך את אסטרטגיית
   הטעימה ב-02 לפי החלטת המוצר).
3. הקמת Stripe (מוצר/מחיר) + פריסת `stripe-webhook` + Checkout שמעביר
   `supabase_user_id` + בדיקת עדכון `subscriptions`.
4. חיווט בקליינט: הפעלת `COMMERCE_LOGIN` כברירת מחדל, גיטינג UI לתוכן
   premium דרך `clHasAccess()`, מסך "שדרג למנוי".
5. **רק עכשיו**: הרצת `03_rls_content.sql` (רגע ה"נעילה") + עדכון כתיבת
   האדמין ל-policy (יש placeholder ל-UUID של האדמין בקובץ).
6. הגנת מדיה: Cloudinary authenticated + פריסת `media-sign` + חיווט
   `clSignedAudio` בנגן (סקיצה ב-`MEDIA.md`).
7. פיילוט תשלום קטן (רכישה/ביטול/כשל/מעבר מכשיר/אופליין) — ואז השקה.

### עבודות תשתית במקביל (לא חוסמות)
- **מודולריזציה הדרגתית** של index.html — רק עם רשת הבדיקות; להתחיל מחילוץ
  תוכן מובנה (BUILTIN_*) לקובצי JSON/JS נפרדים, אחר כך CSS, אחר כך מודולים.
- הפרדת סביבות (preview deployment ב-Cloudflare לפני main).
- delta-sync לתוכן (היום נטען הכול; `cached_phrases` מרכך).
- הרחבת יומן השגיאות לשליחה לשרת (היום מקומי בלבד).

## 5. מוקשים ידועים (לקרוא לפני שנוגעים)
- **אסור** להריץ את `03_rls_content.sql` לפני שלבים 1–4 — תוכן premium ייעלם
  מהאפליקציה החיה לכל המשתמשים.
- כל שינוי ב-index.html מחייב bump ל-`CACHE` ב-sw.js — אחרת משתמשים תקועים
  על גרסה ישנה.
- אין staging: דחיפה ל-main = production מיידי. להריץ `tests/smoke.js` לפני.
- מחרוזות עם גרש בודד `'` בתוך תוכן עברי שוברות JS — התוכן המובנה משתמש
  ב-geresh `׳`; לשמור על זה בכל תוכן חדש.
- ה"קוד גישה" של חבילת האופליין הוא הגנת צד-לקוח בלבד (ידוע ומקובל עד RLS).

## 6. החלטות עסקיות פתוחות (לבעלת המוצר)
1. ספק סליקה (ברירת המחדל בקוד: Stripe).
2. אסטרטגיית טעימה — מה חינם ומה premium (עריכת `02_content_tier.sql`).
3. מחיר ומודל (חודשי/שנתי/ניסיון).
4. שיטת הזדהות (הקוד בנוי לטלפון+OTP; אימייל אפשרי באותה תשתית).

## 7. גישות שיידרשו למפתח
Supabase (dashboard + service role) · Cloudinary · Cloudflare · Stripe ·
GitHub repo. **אף סוד לא שמור בריפו** — המפתח הציבורי בקוד הוא publishable
בכוונה; ה-service role וה-API secrets מוזנים רק כ-secrets של Edge Functions.
